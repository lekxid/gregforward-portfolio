Cloud Infrastructure

My live environment is hosted on Microsoft Azure and structured like a production-style deployment, combining secure public access, containerized services, backend integration, and automated delivery.

Live Production Stack

  • Microsoft Azure Ubuntu Virtual Machine
  • Docker containers for portfolio, backend API, PostgreSQL, and WireGuard
  • Caddy reverse proxy handling HTTPS and domain routing
  • DuckDNS domains for portfolio and VPN access
  • GitHub Actions CI/CD deployment pipeline
  • Public HTTPS access with automatic SSL certificates

Deployment Flow

Code is version-controlled in GitHub and deployed through a repeatable GitHub Actions workflow. The Azure VM runs Docker-based services, while Caddy securely routes incoming requests to the appropriate containers and endpoints.

CI/CD deployment active — updates can be delivered from GitHub to the live server.
